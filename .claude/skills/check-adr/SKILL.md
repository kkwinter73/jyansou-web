# check-adr — 設計準拠チェック

現在の差分（`git diff`）を読み、以下の観点で設計準拠を点検して、違反候補を箇条書きで報告する。**修正は勝手に行わず、指摘に留める**（ターン終了時の総点検向け）。

## 観点

1. **依存方向**（architecture.md）: `core` が `web`/`server`/DOM/React を import していないか。逆流がないか。
2. **core 純粋性**（ADR-0002 / 0006 / 0007）: `Math.random` / `Date` / `fetch` / 破壊的更新 / 可変状態が `core` に紛れていないか。乱数は `RngState` 経由か。
3. **状態遷移**（ADR-0006）: 状態変更が `apply(state, action)` を通っているか。非合法手で例外を投げていないか。
4. **記録の薄さ**（guards.md）: ADR にコード/スキーマで復元できる事実や進捗を書いていないか。decisions-index は再生成済みか。
5. **担当境界**（ADR-0005）: 担当リポ外を変更していないか。

## 出力

- 各違反候補: `ファイル:行` / どのADR・設計に反するか / 是正案（1行）。
- 違反なしなら「設計準拠OK」と該当観点を明記。
